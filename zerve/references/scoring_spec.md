Baseline scoring design

Crime component:
- Derived from Toronto Police Community Safety Indicators and neighbourhood crime rates
- Higher assault, robbery, break and enter, auto theft concentration lowers score

Time component:
- Use reported/occurrence hour distributions where available
- Group into morning, afternoon, evening, night
- Penalize time windows with historically worse incident concentrations

Density component:
- Use as contextual adjustment only
- Dense areas are not automatically unsafe
- Penalize only when combined with elevated incident concentration

Transit component:
- Optional
- Use TTC/community safety indicators where relevant

Confidence:
- Higher when multiple sources agree and data coverage is good
- Lower when cell must inherit nearby geography averages
